# Document Intelligence Refinery Agentic Pipeline

Production-grade document intelligence pipeline with multi-strategy extraction, semantic search, and provenance-backed Q&A.


---

# 🎯 Overview

**Document Intelligence Refinery** automatically processes financial and governmental PDF documents through an intelligent **5-stage pipeline**.

Pipeline stages:

1. Document Upload  
2. Triage & Profiling  
3. Multi-Strategy Extraction  
4. Semantic Chunking & Indexing  
5. Query + Provenance-Backed Answering  

---

# ✨ Key Features

## 🧠 Intelligent Processing

**Multi-signal Triage**

Analyzes document characteristics including:

- Character density
- Image ratio
- Table count

This allows automatic classification of documents.

**Confidence-Gated Escalation**

Extraction automatically escalates between strategies:

```
Strategy A → Strategy B → Strategy C
```

If confidence thresholds are not met.

**4-Stage Processing Pipeline**

```
Triage → Extraction → Chunking → Indexing
```

Each stage provides visible progress feedback.

---

## 🔍 Advanced Retrieval

### Vector Search

- Semantic embeddings stored in **ChromaDB**
- Enables similarity-based retrieval

### Structured Facts

- **SQLite FactTable**
- Optimized for numerical queries

### Hierarchical Navigation

- **PageIndex Trees**
- Enables section-level retrieval and navigation

---

## 💬 Professional Q&A Interface

The system supports **multiple answer modes**.

### Mode Selectable Answers

Users can choose how answers are generated.

#### 🤖 LLM Synthesis
Concise natural-language answers generated by an LLM.

#### 📊 Extract Figures
Regex-based numerical extraction from documents.

#### 🗄️ FactTable Only
Query strictly structured SQL facts.

#### 🔍 Raw Passages
Debug mode showing raw retrieved passages.

---

### Provenance & Auditability

Every answer includes:

- Page numbers
- Source document links
- Retrieval method

**Audit Mode**

Verify claims directly against source documents using spatial citations.

# 🚀 Quick Start

## Prerequisites

- Python **3.11+** (tested on 3.13)
- **Poetry** for dependency management
- **8GB RAM recommended** (4GB minimum)

---

## Installation

```bash
# 1. Clone repository
git clone https://github.com/Addisu-Taye/Document-Intelligence-Refinery-Agentic-Pipeline
cd Document-Intelligence-Refinery-Agentic-Pipeline

# 2. Install dependencies
poetry install

# 3. (Optional) Set OpenAI API key for LLM mode
echo "OPENAI_API_KEY=sk-your-key-here" > .env
```

---

## First Run

1. Open your browser

```
http://localhost:8501
```

2. Upload a **PDF document** in the left panel.

3. Wait for the **4-stage pipeline** to complete:

```
Triage → Extraction → Chunking → Indexing
```

4. Ask questions in the **chat interface**.

5. Toggle the **answer processing mode**.

    self,

    ldu_store: Dict[str, LDU],
    mode: str = "figures"

### Parameters

| Parameter | Description |
|-----------|-------------|
| query | User question |
| doc_id | Document identifier |
| ldu_store | LDU lookup dictionary |
| mode | Answer processing mode |

# ProvenanceChain Schema

```python
class ProvenanceChain(BaseModel):

    query: str
    answer: str
    answer_confidence: float

    citations: List[ProvenanceCitation]

    retrieval_method: Literal[
        "pageindex_navigation",
        "semantic_search",
        "structured_query",
        "hybrid"
    ]
```

# 🐛 Troubleshooting

## Common Issues

| Issue | Solution |
|------|----------|
| ModuleNotFoundError: src | Run commands using `poetry run` |
| ChromaDB error | Delete `.refinery/vector_store` |
| OpenAI API key error | Set `OPENAI_API_KEY` |
| Streamlit port conflict | Use `--server.port 8502` |
| Missing pipeline agents | Check `extracted_document.py` |
